这是一个典型的**状态污染**问题。

### 问题分析
1.  **现象**：用户第一次输入内容后删除（或关闭弹窗），第二次进入时，之前的输入内容似乎还在，甚至自动触发了解析。
2.  **根源**：
    *   `importText` 状态可能没有在每次打开模态框时清空。
    *   更关键的是，原生 `SpeechRecognition` 插件可能在后台保留了上一次的 `partialResults`。
    *   在 `useEffect` 中，当 `showImportModal` 变为 true 时，我们只做了“自动开始录音”，但没有做“状态重置”。

### 修复计划
1.  **强制重置状态**：
    每当 `showImportModal` 变为 `true` 时，强制清空以下状态：
    *   `importText` (输入框文本)
    *   `tempTranscript` (临时语音转录)
    *   `parsedResults` (已解析结果)
    *   `lastImportText` (上一次解析的文本)

2.  **实施位置**：
    修改 `useEffect` 监听 `showImportModal` 的逻辑：
    ```typescript
    useEffect(() => {
      if (showImportModal) {
        // Reset all states
        setImportText('');
        setTempTranscript('');
        setParsedResults([]);
        setLastImportText('');
        
        if (autoStartVoice) {
             // ... existing auto-start logic
        }
      }
    }, [showImportModal, autoStartVoice]);
    ```
