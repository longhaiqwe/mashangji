# 修改版本号为 1.0

这是一个重要的上架准备步骤。将版本号重置为 1.0 通常意味着这是第一个正式发布版本。

## 修改计划

### 1. 修改前端显示的构建版本
*   **文件**: `components/Settings.tsx`
*   **操作**: 将显示的 "Version 1.1.2" 修改为 "Version 1.0"。

### 2. 修改项目配置版本
*   **文件**: `package.json`
*   **操作**: 将 `version` 字段修改为 `1.0.0`。
*   **说明**: 这会影响 `npm run cap:sync` 时同步到原生项目的默认版本。

*   **文件**: `ios/App/App.xcodeproj/project.pbxproj` (虽然可以通过 Xcode 改，但 `package.json` 更关键)
    *   通常 Capacitor 会在 sync 时尝试同步版本，但为了保险起见，建议您在 Xcode 中最终打包前再次确认。我们先改前端和 `package.json`。
